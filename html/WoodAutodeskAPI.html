WoodAutodeskAPI package

Module contents

This is a wrapper for some of the functions on Autodesk Platform Services.

Module is developed for Wood PLC exclusive use.

This module contains tools to interact easily with Autodesk API.

Be mindful, project ids usually have to start with string ‘b.’

Tools in here mainly include data management, such as uploading files to folders, getting all folders inside of a folder, deleting files and obtaining item information.

To start using it, I’ve defined three inputs for the AutodeskTooks class. These are mandatory in order to access all functions of the methods.

check_status_for_publishing(project_id: str, model_id: str) → None | dict

Function which returns a dictionary with information on 3D models available for publishing.

When response is None, there are two options:

  1. There’s not enough permission to access this file.

  2. The 3D model is available for update.

Parameters:

  • project_id (str) –

    Project id Autodesk identifier. Can be obtained by exploring all projects using the API methods, or, easier just using BIM360 and obtaining from the URL.

    This must begin with ‘b.’, else the API won’t be able to access the project.

  • model_id (str) – Model id Autodesk identifier. Can be obtained exploring items using this API wrapper, or, if just interested in one model, can be obtained navigating the BIM360 folder structure.

Returns:

Returns the status of the specified model. If the result is None, the model

is correctly published. If the result is not None, it might be a completed or comitted model. Access the status under ‘data’ ‘attributes’ ‘status’ to learn more about the model.

Return type:

dict | None
